# Codex Review

Target: working tree diff (terminal home refactor)
Date: 2026-06-12

The new terminal home page has a pointer-only lightbox interaction and a visible hero animation regression. These are user-facing issues that should be addressed before considering the patch fully correct.

## Review comments

- **[P2] Make lightbox triggers keyboard-operable** — `src/components/terminal-home/FeaturedWindow.tsx:28`
  For keyboard-only visitors, this screenshot lightbox trigger is just a plain image with `data-lb`, so it is skipped in tab order and cannot be opened with Enter/Space; the archive/media `data-lb` elements use the same pattern. Wrap these triggers in focusable controls or add equivalent `tabIndex` and key handling so the new gallery is not pointer-only.

- **[P3] Start the scramble from unresolved text** — `src/hooks/useHeroText.ts:32`
  When reduced motion is not enabled, this initializes the hero name to the final text, then 600ms later replaces most characters with random glyphs before resolving again. That makes the name flash as already solved before the scramble animation starts; initialize with scrambled/blank rows for the animated path instead.
